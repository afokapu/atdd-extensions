# W6 — Frontend boundaries + train — mirror classification

**Source of record (read-only):**
`/Users/alecfokapu/Github/atdd/main/src/atdd/coder/conventions/frontend.convention.yaml`
(+ its realizing validators under `src/atdd/coder/validators/`:
`route_train_wagon_analyzer.py`, `test_no_stub_presentation_returns.py`,
`test_preact_layer_boundaries.py`).

**Method:** each rule read in full (statement / normative_text / rationale), then
self-classified with quoted evidence. AGNOSTIC-CONSUMER rules are mirrored into the
Vite/React stack (`coder.vite.*`) because every rule in this convention governs a
*consumer* frontend repo's TypeScript/TSX source — and `frg-app`
(`apps/game/src/**`, `plan/_trains.yaml`, `plan/_wagons.yaml`) is exactly such a
consumer that already realizes `<TrainView>`, `FrontendTrainRunner`, the 4-layer
architecture, and the plan registries these rules read.

Target selection: all 12 rules operate on JSX/TSX (`<TrainView trainId=…/>`, functional
component render bodies) and the `web/src/{wagon}/{feature}/{layer}` layout — i.e. the
Vite/React stack. So every agnostic rule mirrors into `frontend.extension.vite-coder`
+ `frontend.workspace.runtime`. (The Astro stack ships `.astro` components with a
frontmatter fence, not React functional components or `<TrainView>` routers, so these
rules do not have an Astro realization.)

Legend: **A** = AGNOSTIC-CONSUMER (mirror) · **I** = ATDD-INTERNAL (why-not).

`plan/_wagons.yaml`. The plan registries are consumer-owned artifacts here, not ATDD
substrate → the whole family is a consumer obligation with a TSX detector → **MIRROR**.

Rule 003 stays advisory (`severity: 1`, "advisory; never hard-fail") in the mirror,
preserving Decision #4 ("Hard-fail suppressed").

property of the detector (a guarded-null component yields **zero** violations). It is
therefore NOT listed in the detector's `emits_rule_ids` (it never emits); the
guarantee is verified by the `test_W6.py` negative assertion and by the family's
`fixtures/clean/…/GuardedNull.tsx`.

## Not classified as separate mirror rules (subsumed / substrate)

These appear in `frontend.convention.yaml` but are **not** in the W6 rule list and are
recorded here so nothing is silently dropped:

- **`frontend.layers` catalog** (component_types / suffix / structure, lines 5–514):
  a descriptive catalog of layer→suffix mappings, not an enforceable rule_id. It has
  no `rules:` id and no disposition; it is reference material consumed by the
  boundaries validators above. **Not a standalone rule → subsumed into
  `boundaries-fe-layers`.**
- **`http_access` / `no_raw_fetch`** (SPEC-CODER-CONTRACT-0001, lines 539–565): **A**,
  but **already mirrored** as `coder.vite.boundaries-http-client`
  (`vite_boundaries_http_client`). Out of W6 scope by the assignment ("http-client
  done"). Not re-built.
- **`train_composition` page-elimination specs** (SPEC-CODER-PAGE-0001/0003/0004,
  SPEC-CODER-TRAIN-0001/0002/0003): these are `SPEC-*` config-driven structural specs
  (`page_elimination`, `god_hook_elimination`, `frontend_composition_root`,
  `wagon_trains_export_shape`, `train_yaml_render_metadata`). They are **not in the W6
  rule list** (they are covered by the Train / composition-root workers), and several
  are **SUBSTRATE-SPEC** ids governing `.atdd/config.yaml`-declared file existence
  rather than a per-line source detector. **Documented, not mirrored by W6.**
- **`ci_enforcement` / `tools` / `failure_policy`** (lines 664–678): CI wiring
  metadata (tool names, `block_on_violation`), **ATDD-INTERNAL** pipeline config, no
  per-consumer source obligation → **not mirrored.**
