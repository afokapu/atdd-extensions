## AGNOSTIC-CONSUMER — mirrored (built)

Realizations live in `official/convex.workspace.runtime/implementations/` (detectors)
and `official/convex.extension.tester/conventions/` (nodes). The three smoke rules
share ONE FAMILY validator (`convex_smoke_fidelity_detector`, Core FAMILY pattern);
red + routing are singletons.

### 1 — `tester.smoke.no-collaborator-substitution` → `tester.convex.smoke-no-collaborator-substitution`
Evidence the obligation is agnostic and the TS realization is the intended follow-up
(quoted from `smoke.convention.yaml::collaborator_substitution_rules`):

> "A SMOKE test must exercise the real subject. A test that substitutes one of the
> subject's collaborators … is a unit test wearing a SMOKE label: it passes CI while
> exercising nothing real."

> "Scope is Python-only; TS smoke tests (consumer-repo frontend e2e) are a #704
> follow-up."

The obligation applies to any consumer's smoke tests; only the substitution *syntax*
(`vi.fn`/`vi.mock`/`jest.*`/`.mockImplementation`/attribute-reassignment) is
stack-specific → a Convex/Vitest detector. FAMILY member
`checks/smoke_no_collaborator_substitution.mjs`. severity 4, suppress-and-clean.

### 2 — `tester.smoke.operator-observable-assertion` → `tester.convex.smoke-observable-assertion`
Evidence (quoted):

> "SMOKE tests MUST assert on operator-observable behavior … not on
> intermediate-artifact-write events".

and the convention's decidable per-stack check (`validation.checks.db_assertion_present`):

> "Smoke tests that mutate state must verify database … Smoke tests with
> POST/PUT/PATCH/DELETE calls should include DB queries"

and the anti-pattern (`forbidden_patterns.response_only_assertions`):

> "No database query after a POST/PUT/PATCH/DELETE call".

Agnostic: a state-mutating smoke in any consumer must assert the persisted result.
The Convex/TS realization flags a smoke that mutates (`.post/.put/.patch/.delete`,
`.mutation(`, `runMutation(`) with no observable read-back (`.query/runQuery/.select/
.get/.collect/.first/.unique`). FAMILY member `checks/smoke_observable_assertion.mjs`.
severity 3, documentation-only.

### 3 — `tester.smoke.pres` (TESTER-SMOKE-PRES-001) → `tester.convex.smoke-presentation-coverage`
Evidence (quoted from `presentation_smoke_rules` / `validation.checks.presentation_smoke_coverage`):

> "Every web/src/*/presentation/*.tsx must have a sibling smoke test under
> e2e/*smoke*.spec.ts"

> "For each presentation file under web/src/<wagon>/presentation/*.tsx, confirm at
> least one e2e/**/*smoke*.spec.ts whose path contains the wagon name as a
> kebab/snake/word-boundary token."

Agnostic frontend-coverage obligation; the file-shape is the per-stack rendering. The
Convex/TS realization pairs each presentation `.tsx` with a `*smoke*.spec.ts` carrying
its wagon token. FAMILY member `checks/smoke_presentation_coverage.mjs`. severity 3,
documentation-only.

### 4 — `tester.red.fails-first` (TESTER-RED-FAILS-FIRST-001) → `tester.convex.red-fails-first`
Evidence: the rule (quoted) —

> "RED tests must fail on first run before implementation begins"

is a runtime property, but `red.convention.yaml` supplies the decidable STRUCTURAL
proxy: `validation_levels.2_structure` requires "RED marker present", and
`red_patterns.typescript` enumerates the guaranteed-fail markers
(`throw new Error('Not implemented: ${URN}')`, `expect(false).toBe(true)`,
`return Promise.reject(new Error('${URN}'))`). Agnostic; the marker syntax is
per-stack. Singleton detector `convex_red_fails_first`. severity 3, documentation-only.

### 5 — `tester.routing.path` (TESTER-ROUTING-PATH-001) → `tester.convex.routing-path`
Evidence (quoted):

> "Test path determines the runtime that executes it (python/ vs supabase/ vs web/)"

and the principle (`runtime_placement`):

> "Test path determines implementation runtime".

Agnostic invariant: a test's location must agree with the runtime that runs it. The
Convex/TS realization flags a `*.test.ts` (run by Vitest/node) whose `Runtime:` header
names a non-TypeScript runtime (`python`/`dart`/`go`/…) — the path contradicts the
declared runtime. Singleton detector `convex_routing_path`. severity 2,
documentation-only.

## Documented WHY-NOT (not mirrored)

### 7 — `tester.smoke.cross-component-handoff-gap`
Tester-owned, documentation-only. The obligation ("when a feature has a
producer→consumer handoff, the SMOKE MUST exercise both ends in a single test") is
agnostic in intent, but its `handoff_detection` (quoted) —

> "producer: component that writes to a shared artifact (cli-return.jsonl, output.log,
> queue) — consumer: component that reads and acts on that shared artifact — required:
> single SMOKE test that asserts on both producer write AND consumer reaction"

— requires knowing WHICH features have a producer→consumer seam. That is a
feature-topology fact from the plan graph, not decidable from a single consumer TS
test file. There is no sound static per-stack detector; realization belongs to
plan-analysis time, not a file scanner. **Not mirrored** (no false-precision detector).

### 8 / 9 — `tester.smoke.train-mounts-but-the` (RENDER-001), `…-1` (RENDER-002)
Both are RUNTIME classifications of a harness result (`behavioral_render`), not static
source checks. Quoted:

> "For every train in plan/_trains.yaml, mount the train through
> FrontendTrainRunner.runTrain in a headless harness and assert the resulting DOM is
> non-empty and not a stub-only render."

> harness runtime: "node + @testing-library/preact + jsdom (vitest)" … invocation:
> "node .atdd/harness/mount-train.mjs --train <train_id>"

Empty-render / stub-render are runtime-behaviour properties of a mounted train —
undecidable from static source (the convention itself invokes Rice's theorem for the
sibling smoke rule). Our v1.1 detector contract is a static file scanner
(`ATDD_SCAN_ROOTS` → RAW report), which cannot mount trains. **Not mirrored as a
static detector**; the realization is a runtime render-harness gate (a distinct
delivery vehicle outside this wave's static-detector scope). The static-source stub
peer (#334 `detect_stub_presentation_returns`) is a *coder* concern, not tester.

### 10 — `tester.smoke.harness-subprocess-failed-crash` (RENDER-003)
Quoted: "Harness subprocess failed (crash, timeout, or schema-invalid stdout) when
mounting the train." This classifies a failure of the ATDD render-harness subprocess
itself (`.atdd/harness/mount-train.mjs`, `train-render-harness-result.schema.json`) —
an ATDD-substrate artifact, not consumer application code. **Not mirrored** (governs
the toolkit's own harness, per SHARED METHOD's SUBSTRATE-SPEC class).

### 11 — `planner.smoke.synthetic-fixture-bypass`
PLANNER-OWNED by construction: the `planner.` rule_id prefix and the validator path
(quoted) —

> "Planner-side validator:
> src/atdd/planner/validators/test_smoke_synthetic_fixture_bypass.py"

place it in the planner layer (it governs whether a *plan's* WMBT smoke drives the real
CLI entry point, `atdd spawn` / `atdd coach`). Not a consumer per-stack code check.
**Not mirrored** (planner-owned).

### 12 — `planner.smoke.feedback-loop-close-the-loop`
PLANNER-OWNED: `planner.` prefix; validator (quoted)
`test_feedback_loop_smoke_closes_the_loop::test_every_feedback_loop_feature_has_close_the_loop_smoke`.
It asserts a *feature YAML* marked `kind: feedback-loop` has a WMBT SMOKE acceptance
with a `close_the_loop:` block — a plan-layer acceptance-structure obligation over
`plan/<wagon>/features/*.yaml`, not consumer application/test code. **Not mirrored**
(planner-owned).
