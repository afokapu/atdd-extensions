# W5 classification — `coder/conventions/green.convention.yaml` (GREEN phase / URN traceability)

Source (read-only): `/Users/alecfokapu/Github/atdd/main/src/atdd/coder/conventions/green.convention.yaml`
(`schema_version: 1.0.1`, `convention_id: coder.green`).

Method: for every rule_id / structured id in the convention I self-classify
**AGNOSTIC-CONSUMER** (obligation applies to any consumer repo's code — a Convex/TS
app is a consumer — so **MIRROR** with a per-stack detector) vs
**ATDD-INTERNAL / PLANNER-OWNED / PROCESS-GUIDANCE / OWNED-ELSEWHERE**
(**DOCUMENT WHY-NOT**, quoting the proving text). Conservative rule: a convention worded as
a per-file obligation with TypeScript/frontend/backend variants is AGNOSTIC.

The convention's own text scopes these rules to **consumer implementation files**:
`file_header_requirements.description: "All implementation files MUST include traceability
markers in file header"` and `component_urn_marker: required: true … "Every implementation
file MUST have component URN marker"`. The URN/header family is therefore AGNOSTIC and is
mirrored into BOTH backend (Convex) and frontend (Vite) — it is stack-neutral (`side:
[frontend, backend]`), so per method §3 it is mirrored into both.

**Note on GREEN-URN-003 (manifest cross-reference).** The full obligation has two halves:
(a) the segments are *present and well-formed* (consumer-detectable from the file alone) and
(b) they *exist in the wagon manifest* (requires the consumer's plan/manifest — a coach/graph
join, not a source scan). This is not a silent drop: the mirrored detector realizes half (a)
faithfully (kebab-case wagon+feature segments) and the fix_hint/normative_text state that the
manifest-existence join is enforced downstream by the consumer's coach against its own manifest,
exactly as the source says (`source: "Extracted from wagon manifest plan/{wagon}/_{wagon}.yaml"`).

## ATDD-INTERNAL / PLANNER-OWNED / PROCESS-GUIDANCE → DOCUMENTED (not mirrored)

### Green-phase *principles* — process guidance, not per-file detectable obligations
- **GP-01..GP-04** — `"Do minimum to satisfy behavior"`, `"Defer structure and optimizations"`,
  `"Avoid irreversible coupling"`, `"Prefer duplication over premature abstraction"`. These are
  phase philosophy for the coder *actor*, not machine-checkable obligations on a consumer file
  (there is no `checks:` block, no per-file predicate). PROCESS-GUIDANCE → not mirrored.
- **GP-05** — `"Define HOW through schema-driven architecture: validate all external data against
  schemas"`. Realized (where automatable) by guardrail **GR-SCHEMA**; the principle itself is
  advisory. Not separately mirrored.

### Guardrails owned by other slices, or non-automatable
- **GR-BASICSEC** — `"Basic security sanity checks (no secrets in code…)"`, checks
  `password\s*=\s*['"]…`, `api_key…`, `secret…`, `Bearer …`. **OWNED-ELSEWHERE**: already realized
  by `coder.convex.security-hardcoded-secret` (per MIRROR-GAP-ASSIGNMENT W8: "hardcoded-secret
  already done — skip"). Not re-mirrored to keep each rule_id single-owner.
- **GR-PORTS** — `"All side-effects MUST be behind a tiny port/interface"`, checks scoped to
  `presentation/**` and `domain/**` layer paths. AGNOSTIC in nature but **OWNED-ELSEWHERE** by
  the backend-architecture / boundaries slice (W3/W6; cf. existing `coder.convex.domain-no-convex-import`,
  `coder.convex.layer-naming`). Classified AGNOSTIC-but-deferred to avoid a duplicate rule_id.
- **GR-SCHEMA** — `"All external data MUST be validated against schemas"`, checks
  `must_match: [zod|z.object|…]` in `presentation/**`. AGNOSTIC in nature but a
  boundary/validation concern that overlaps the boundaries/DTO slice (W3). Classified
  AGNOSTIC-but-deferred (single-owner rule_id discipline); not mirrored in the W5 green slice.
- **GR-PURE** — `"Core decision logic MUST be in a pure function"`, whose only check is
  `type: reference, note: "Manually verify core logic is extractable and testable without mocks"`.
  NON-AUTOMATABLE by construction (manual reference check) → not mirrored.

### Shortcuts / deliverables / done-criteria / anti-patterns — process artifacts
- **SH-FLAT, SH-INMEM, SH-DUP, SH-FILES** — each carries `permitted: true`; these are *allowances*
  ("Handler can call gateway/repo directly", "Use in-memory fakes", "Duplicate until 3rd
  occurrence", "Flat files/dirs"), i.e. the opposite of an obligation. Nothing to enforce → PROCESS.
- **DL-PASS** (`"All acceptance tests must pass"`, `type: test_suite`), **DL-ENTRY, DL-PORTS,
  DL-TODOS** — phase deliverables/checklists, verified by running the suite / manual review, not a
  static consumer-file scan. PROCESS-GUIDANCE. (DL-PASS is the tester/CI gate, not a coder-file rule.)
- **DC-TESTS, DC-SEAMS, DC-PORTS, DC-PURITY** — `done_criteria` (handoff checklist to REFACTOR) →
  PLANNER/PROCESS, not per-file detectors.
- **AP-OPT, AP-ABST, AP-FWCOUPLE, AP-MISSINGPORTS** and the `runtime_placement.anti_patterns`
  (AP-WRONGRUNTIME, AP-NONCOLOCATION, AP-NOTEST) — advisory anti-patterns / phase guidance;
  the port-related ones restate GR-PORTS (owned-elsewhere). Not mirrored.

### Structural / substrate sections
- **`runtime_placement`** (`"Implementation path follows test path"`, python/supabase path
  derivation) — a planner/structural colocation concern tied to the ATDD test/src layout; partly
  realized by existing `coder.convex.feature-layout-promotion`. PLANNER-OWNED / OWNED-ELSEWHERE.
- **`composition_root`** (`composition.py` / `wagon.py` DIRTY-GLUE wiring) — already realized by
  existing `coder.convex.composition-root` and `coder.convex.composition-consumer`. OWNED-ELSEWHERE
  (done), not re-mirrored.
- **`ci_gates`** (`on: pull_request`, `require: [GR-PORTS, GR-NOGLOBALS, …, DL-PASS]`) — a CI wiring
  manifest referencing the ids above; not itself a rule. Orchestrator/CI concern.
