# W1 — Interlocking mirror classification

Source (read-only): `atdd-extensions/official/atdd.extension.train-interlocking-enforcement/`.
Method: for each rule_id, READ the convention `statement` / `normative_text`, classify
**AGNOSTIC-CONSUMER** (the obligation binds *any* consumer repo's code; a Convex/TS app is a
consumer → MIRROR) vs **ATDD-INTERNAL / PLANNER-OWNED / SUBSTRATE-SPEC** (governs only the ATDD
toolkit's own substrate or a plan-layer spec id → DOCUMENT WHY-NOT). Evidence is quoted from the
source node.

**Result: all 10 are AGNOSTIC-CONSUMER → all mirrored.** Every convention is worded as an *agnostic
obligation* on a consumer runtime/test tree, with an explicitly *per-stack* python-pytest detector as
"the implementation referenced below" — the textbook mirror shape. None govern ATDD substrate; none
carry a `SPEC-*/AP-*/GP-*/DS-*` internal id. The core-design refs (`afokapu/atdd#1246/#1248/#1251`)
are referenced NARRATIVELY (boundary spec §6.2), not as substrate this extension owns.

## ATDD-INTERNAL / PLANNER-OWNED / SUBSTRATE-SPEC (why-not)

**None.** No W1 convention governs the ATDD toolkit's own substrate, a planner/plan-layer concern, or
a `SPEC-*/AP-*/GP-*/DS-*` internal spec id. The interlocking YAML *schema* (route-space shape,
`entrypoint` field) IS owned by core `afokapu/atdd#1248/#1249` — but bilateral-binding explicitly
**CONSUMES** that field to validate a consumer tree and forbids forking it ("this extension defines
NO parallel reachability field"), so the binding obligation itself is a consumer obligation, not
substrate ownership. Mirrored accordingly.

## Mirror target rationale

Interlocking is a **backend route-control runtime** (Python `python/trains/**`, `python/app.py`,
`plan/_trains/**`). Its stack-neutral analog in a Convex/TS consumer is the Convex backend
(`convex/trains/**`, `convex/app.ts` composition root, `convex/<wagon>/wagon.ts`), grounded against
`frg-app/apps/game/convex/` which already uses the `convex/<wagon>/wagon.ts` + `TrainRunner` +
composition-root shape. It is a server-side concern, so it mirrors into the **convex** backend
(coder) + **convex tester** packages only — NOT the vite/astro frontend packages (no route-control
runtime lives there).
