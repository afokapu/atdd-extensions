## Framing — why acceptance-violation + coverage are overwhelmingly SUBSTRATE-SPEC

Both conventions govern **ATDD's own acceptance/plan substrate** — the `plan/`
acceptance YAML, `feature.yaml` security blocks, `.atdd/metrics/*.py` compute
modules, telemetry `_tracking_manifest.yaml`, and the artifact-coverage graph.
That substrate has an **identical format on every stack** (a Convex/TS consumer
authors the very same acceptance YAML as a Python consumer). The mirror wave's
realization targets are per-*stack* detectors over per-*stack* **code**
(`coder.convex.*`, `tester.convex.*`). A validator that parses acceptance YAML is
**not** per-stack — there is no "Convex sibling" of "every acceptance must declare
`identity.phase`", because acceptance YAML is not Convex source. Those rules are
therefore SUBSTRATE-SPEC / PLANNER-OWNED and are documented why-not, not mirrored.

The **one** exception is `live-smoke-acceptance-must-execute`: its failure mode
and fix live in **consumer test CODE** (a test that can *self-skip*), and its fix
text enumerates **per-stack test-framework primitives** (`pytest.skip` /
`pytest.importorskip` / `@pytest.mark.skipif`). That is the tell of an AGNOSTIC
obligation with a per-stack detector — mirrored below as the Convex/Vitest sibling
`tester.convex.live-smoke-no-self-skip`.
